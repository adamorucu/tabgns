Automated Model Design using Gated Neuron Selection in Telecom

Adam Orucu, Marcus Medhage, Farnaz Moradi, Andreas Johnsson, Sarunas Girdzijauskas
Ericsson Research, KTH Royal Institute of Technology
NOMS 2026
Paper Code arXiv

Workflow of the operational steps in the ML model lifecycle management augmented with TabGNS for automated neural architecture search and training.

Summary

AI-native telecom networks rely heavily on neural networks. Designing models that are both compact and efficient remains a key challenge. We introduce TabGNS, a method that automates the design of accurate, lightweight models.

The resulting architectures match or exceed state-of-the-art prediction performance while using up to 85% fewer parameters and achieving up to 36× faster search. TabGNS addresses the challenge of automating neural network design for tabular telecom applications where the manual process of architecture selection has been both time-consuming and suboptimal because of the need to balance prediction performance and resource constraints.

Why Neural Architecture Search?

Neural networks are widely used in telecom, but designing them is time-consuming and requires expert knowledge. Neural Architecture Search (NAS) automates this process using learning algorithms to discover effective architectures. We introduce TabGNS, a novel NAS method tailored for MLPs, and demonstrate its strong performance on tabular data in telecom use cases.

Gated Neuron Selection

TabGNS introduces gates for each neuron, allowing the model to learn which neurons to include in the final architecture. This enables gradient-based optimization to efficiently discover high-performing MLPs. TabGNS starts the search from small architectures and progressively grows them, which makes it select to small yet accurate architectures.
